# ITC2019 MAZE 实验计划表

本文档记录当前项目从 Phase 1 class assignment 到 Phase 2 student assignment 的完整实验计划，并标注目前完成状态。

状态说明：

| 状态 | 含义 |
|---|---|
| 已完成 | 已有可引用结果或报告 |
| 部分完成 | 已有初步结果，但还缺全实例、多 seed、官方验证或消融 |
| 待完成 | 还没有系统实验结果 |

## 总体 Pipeline

```text
ITC2019 raw data XML
 -> SOTA/DTU reduced data XML
 -> Phase 1 MIP class assignment
 -> Phase 1 solution pool
 -> Phase 1 post-optimization
 -> optimized class solution
 -> Phase 2 MIP student assignment
 -> Phase 2 student LNS / MARL optimization
 -> final complete solution XML
 -> official validator
```

## Phase 1 覆盖范围说明

ITC2019 原始数据约有 30 个 instances。目前项目中只有约 15 个 instances 拥有可用的 MIP solution pool，因此 Phase 1 目前只能算完成了一半。

剩余 instances 的主要困难是约束对数量过大，完整 MIP 在之前的服务器上无法稳定跑完。项目中曾尝试通过图分解降低问题规模，但图分解方案得到的解质量不如完整 MIP，因此暂时不能作为最终 Phase 1 baseline 的替代。

因此论文中需要明确区分两类实验范围：

| 范围 | 含义 |
|---|---|
| Available-pool instances | 当前已有 MIP solution pool 的约 15 个 instances，可用于 Phase 1 post-optimization 和 Phase 2 student assignment |
| Full ITC2019 instances | 全部约 30 个 instances，其中剩余大规模实例仍缺完整 MIP solution pool |

后续如果有更强服务器，应优先补齐剩余 instances 的 Phase 1 MIP solution pool。若仍无法完整求解，则需要把图分解作为一个单独 baseline 报告，而不是直接与完整 MIP pool 混合比较。

## 当前结论

Phase 1 目前只覆盖约一半 ITC2019 instances，即已有 MIP solution pool 的 instances。已有结果说明 post-optimization 可以在部分可用实例上改进 MIP pool，尤其是 invalid pool 较高的实例。下一步有两个关键方向：一是补齐剩余 instances 的 Phase 1 MIP pool，二是在已有 Phase 1 优化后 solution 上补齐 Phase 2 student assignment，并用官方 validator 验证最终完整 XML。
